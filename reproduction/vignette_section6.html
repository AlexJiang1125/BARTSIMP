This vignette reproduces the application in Section 6 of

Jiang, A. Z., & Wakefield, J. (2025). BARTSIMP: Flexible spatial covariate modeling and prediction using Bayesian Additive Regression Trees. Spatial and Spatio-temporal Epidemiology, 55, 100757. https://doi.org/10.1016/j.sste.2025.100757

Section 6 fits BARTSIMP to child wasting (weight-for-height Z-score, WHZ) from the 2014 Kenya DHS with six geospatial covariates, and does three things:

  1. Stratified cross-validation that splits clusters (enumeration areas), comparing BARTSIMP against BART / SPDE / SPDE0 on RMSE / AIL / ACR / AIS.
  2. Population-weighted areal aggregation of the point-level surface to administrative areas, with credible intervals from the areal posterior.
  3. Partial dependence to read off each covariate’s marginal effect on WHZ.

Data note. The DHS microdata are not redistributable — you must register with the DHS Program and download them yourself. So this vignette runs on a synthetic Kenya-DHS-like dataset with the same structure (clusters, strata, six named covariates, nested admin areas, under-five population weights). It reproduces the workflow, not the paper’s numbers. reproduction/R/covariates_kenya.R documents exactly how to assemble the real inputs; once you have them in the same column layout, every code chunk below runs unchanged.

4 Analysis 1: stratified cluster cross-validation

The paper splits clusters (not children) into train/test, preserving the stratified design (county x urban/rural). Splitting by cluster is essential: children in one cluster share a location, so a row-wise split would leak spatial information into the test set. We score each method against the held-out cluster-mean WHZ (averaging a cluster’s test children removes most child-level noise, so the mean-field intervals returned by the fitters are the right object to score).

cv <- run_cv_section6(
  sim, reps = 1:2, methods = c("BARTSIMP", "BART", "SPDE", "SPDE0"),
  prop_train = 0.8, alpha = 0.05,
  bartsimp_args = list(ntree = 30L, ndpost = 150L, nskip = 1L, nwarmup = 150L),
  verbose = FALSE)
knitr::kable(summarize_cv_section6(cv)[, c("method", "RMSE", "AIL", "ACR", "AIS")],
             digits = 3, row.names = FALSE,
             caption = "Cluster CV, 2 repeats (means). Nominal coverage 0.95.")

Interpretation mirrors the paper: BARTSIMP is competitive on RMSE while keeping intervals honest, because it captures the covariate signal (BART) and the residual spatial structure (Matérn field). These are tiny-run numbers — ndpost = 150, 2 repeats — so coverage is not yet calibrated; raise the MCMC size and reps toward the paper settings (last section) before reading anything into the ranking.

5 Analysis 2: population-weighted areal estimates

For small-area estimation the paper aggregates the point-level surface to administrative areas with an under-five population-weighted average, \[ \mathrm{WHZ}_R = \frac{\sum_{g \in R} \mathrm{WHZ}(s_g)\, d_5(s_g)} {\sum_{g \in R} d_5(s_g)}, \] and takes the area’s credible interval from the areal posterior (apply the weighted average to every posterior draw, then take quantiles) — not by combining cell-level intervals. fit_bartsimp(..., return_draws = TRUE) returns the full cell-level posterior for exactly this purpose.

fit <- fit_bartsimp(obs, grid, cov_names = SECTION6_COVARIATES,
                    ntree = 30L, ndpost = 250L, nskip = 1L, nwarmup = 250L,
                    alpha = 0.05, return_draws = TRUE)

truth    <- true_areal(grid, level = "admin1", weight = "pop_u5")

The model-based areal estimate borrows strength across space and covariates, so it typically tracks the truth far better than a direct estimate that uses only each area’s own clusters — the qualitative point of the paper’s small-area maps. (Coverage again needs a larger MCMC run to calibrate.)

6 Analysis 3: partial dependence

Partial dependence reads off the marginal effect of a covariate on WHZ, holding the others fixed — the paper uses it to interpret the fitted covariate model. Here is a self-contained partial-dependence computation on the BART covariate mean (via dbarts with keeptrees = TRUE), which recovers the signs built into the synthetic covariate effect (WHZ rises with population density and precipitation, falls with temperature and travel time to a city).

set.seed(1)
Xtr <- as.matrix(obs[, SECTION6_COVARIATES, drop = FALSE])
bfit <- dbarts::bart(x.train = Xtr, y.train = obs$y, keeptrees = TRUE,
                     ntree = 50L, ndpost = 400L, nskip = 400L, verbose = FALSE)

partial_dependence <- function(fit, X, var, n_grid = 20L, n_bg = 250L) {
  bg   <- X[sample(nrow(X), min(n_bg, nrow(X))), , drop = FALSE]
  grid <- seq(quantile(X[, var], 0.05), quantile(X[, var], 0.95),
              length.out = n_grid)
  mm <- vapply(grid, function(g) {
    nb <- bg; nb[, var] <- g
    mean(colMeans(predict(fit, nb)))   # average over draws and background rows
  }, numeric(1))
  data.frame(x = grid, whz = mm)
}

op <- par(mfrow = c(1, 2), mar = c(4, 4, 2, 1))
for (v in c("pop_density", "temp")) {
  pd <- partial_dependence(bfit, Xtr, v)
  plot(pd$x, pd$whz, type = "l", lwd = 2, col = "#1f6fbf",
       xlab = v, ylab = "partial dependence (WHZ)",
       main = paste("PDP:", v))
}

par(op)

Population density shows a positive partial effect and temperature a negative one, matching the data-generating .covariate_effect(). On real data these curves are the interpretive payoff of the flexible BART mean — non-linearities that a linear SPDE covariate term would miss.

7 Reproducing the paper-scale application

To approach the paper, with the real DHS data assembled via covariates_kenya.R:

  • Use all 1584 clusters and the six real covariate rasters; keep the county x urban/rural strata.
  • CV: run_cv_section6(sim_like, reps = 1:10, prop_train = 1267/1584) with BARTSIMP ndpost = 1000, nwarmup = 1000.
  • Areal: fit_bartsimp(obs, grid, return_draws = TRUE) on a fine national grid, then aggregate_areal(..., level = "admin1") and "admin2".
  • Score per-child with fully predictive intervals if you want the paper’s exact numbers (add a \(\sigma_e\) inflation to the mean-field intervals); the method ranking is unchanged.

Runtime. Stage-2 prediction uses the Woodbury closed form by default (see the Section 5 vignette), so it is cheap; the C++ sampler (Stage 1), which calls INLA once per MCMC iteration, dominates a full run. Budget an overnight job for the paper-scale CV sweep.
